# GEMINI.md

Guidance for Gemini CLI when working in this repository.

## Project

HackUPC 2026 entry: a peer-to-peer compute exchange where peers trade LLM inference time for local credits. QVAC SDK handles local/delegated inference and the QVAC Hyperswarm transport. This repo layers provider discovery, credit accounting, a CLI, and an OpenAI/Ollama-compatible local HTTP proxy on top.

The main product surface is:

- Provider runtime: `pear run . serve`
- Consumer HTTP proxy: `pear run scripts/server.js`, listening on `http://127.0.0.1:11434`
- CLI tooling: `pear run . <command>`

## Commands

Run app entrypoints through Pear/Bare.

- `pear run . serve [--models keys] [--peer-name name] [--topic hex] [--skip-download]` starts provider mode. It pre-downloads served models, starts QVAC provider mode, joins discovery, advertises served models, and earns credits from `creditAck` messages.
- `pear run scripts/server.js` starts the working HTTP proxy. It exposes `/api/chat`, `/v1/chat/completions`, `/api/peers`, `/api/balance`, and `/api/version`, then delegates chat requests to discovered providers.
- `pear run . daemon [--host addr] [--port n]` starts the generic API shell. It wires peers and balance, but does not currently wire delegated chat. Use `pear run scripts/server.js` for chat.
- `pear run . ask [--model key] [--peer id] [--max-credits n] [--strategy cheapest|best|fastest|rated] <prompt>` delegates a prompt from the CLI. It streams output but does not currently spend credits or send `creditAck`.
- `pear run . peers` reads discovered peers from the local daemon/server.
- `pear run . balance` reads local ledger state from the local daemon/server.
- `pear run . rate <provider-id> <1-5>` calls `/api/rate`, which currently returns `501`.
- `pear run scripts/local-test.js` validates local QVAC load and completion.
- `pear run scripts/delegated-test.js` runs provider plus known-key consumer smoke test.
- `pear run scripts/discovery-test.js` runs discovery-only smoke test.
- `pear run scripts/auto-consumer.js` discovers one provider, delegates one prompt, spends credits, and sends `creditAck`.
- `pear run scripts/e2e-test.js` resets `data/`, runs provider plus auto-consumer, and checks ledger symmetry.

Demo:

```bash
PEER_NAME=alice pear run . serve
PEER_NAME=bob pear run scripts/server.js
curl -X POST http://127.0.0.1:11434/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"llama-1b","messages":[{"role":"user","content":"Say hi"}]}'
```

## Architecture

Core layers:

1. `src/core/qvac.js` wraps `@qvac/sdk`. Keep direct SDK imports out of the rest of the codebase.
2. `src/core/discovery.js` is a separate Hyperswarm side channel. It announces peer name, served models, QVAC topic, and QVAC provider public key. It also carries trust-based `creditAck` messages.
3. `src/core/ledger.js` stores a local JSON ledger in `data/<peerName>.ledger.json`.
4. `src/server/provider-runtime.js` starts provider mode, pre-downloads model assets, wires discovery, and applies earned credits.
5. `src/server/compute-exchange-api.js` implements HTTP route handling. The caller supplies `onChat`, `onGetPeers`, and `onGetBalance`.
6. `scripts/server.js` is the working HTTP chat bridge. It wires discovery, ledger, delegated QVAC calls, streaming responses, spend entries, and an attempted `creditAck`.
7. `cli/commands.js` implements the Pear CLI. Some CLI paths duplicate server behavior; keep this in mind when changing API contracts.

Configured models live in `src/config.js`:

- `llama-1b`: `LLAMA_3_2_1B_INST_Q4_0`, tier `1`
- `qwen-1.7b`: `QWEN3_1_7B_INST_Q4`, tier `3`

Topics live in `src/topics.js` and are generated by `sha256(name)` to satisfy Hyperswarm's 32-byte topic requirement.

## Built vs Placeholder

Built:

- QVAC wrapper
- Provider runtime
- Discovery announces and peer tracking
- `creditAck` handling
- Local JSON ledger
- Multi-model catalog
- OpenAI SSE streaming on `/v1/chat/completions`
- Ollama NDJSON streaming on `/api/chat`
- Peer and balance API routes
- Provider, server, manual consumer, auto-consumer, and smoke-test scripts

Placeholders or partial:

- `/api/tags` returns an empty model list placeholder.
- `/api/generate` returns not implemented.
- `/api/rate` returns `501`; ratings are not persisted.
- `pear run . daemon` does not wire `onChat`.
- `pear run . ask` does not currently debit credits or send `creditAck`.
- `scripts/server.js` spends credits locally but currently sends `creditAck` to `provider.qvacProviderPublicKey`; `Discovery.sendCreditAck` expects an active discovery peer id. Audit this before relying on provider earn updates through the HTTP path.
- No signed receipts, shared ledger, consensus, load balancing, or real provider reputation.

## Non-obvious Constraints

- Hyperswarm topics must be 64 hex chars. Short topics can appear to start successfully but delegated consumers time out with `DELEGATE_CONNECTION_FAILED`.
- Provider and delegated consumer should not run in the same process. QVAC worker/RPC behavior is effectively process-singleton for this flow.
- Bare does not expose `process` globally. Entry scripts that use env/stdout/signals need `import process from "bare-process"`. Use `bare-os` for hostname.
- Pear needs generated QVAC worker assets. `qvac.config.json` drives `npx qvac bundle sdk`, which generates `qvac/worker.entry.mjs` and `qvac/addons.manifest.json`.
- Import `qvac/worker.entry.mjs` before code that calls the QVAC SDK.
- `qvac/worker.bundle.js` is generated for Expo/RN, gitignored, and should not be committed.
- DHT bootstrap can take 10-30 seconds and can fail on restrictive networks.

## Editing Guidance

- Add models in `src/config.js`; do not inline SDK constants in scripts.
- Keep QVAC SDK usage behind `src/core/qvac.js`.
- If changing route behavior, check both `src/server/compute-exchange-api.js` and the callers in `scripts/server.js` and `cli/commands.js`.
- If changing provider startup, prefer `src/server/provider-runtime.js` so `scripts/provider.js` and `cli serve` stay aligned.
- If changing credit pricing, update `src/ledger-config.js` and remove any duplicated formula in callers.
- If changing `qvac.config.json` or the SDK version, regenerate QVAC worker assets.
